## Issues to Fix (Priority Order)

### Issue 1: PreToolUse Edit|Write Hook — 27% Denial Rate [CRITICAL]

**Problem:** The prompt-type hook blocks edits when it can't verify sequential files. Circular dependency on CLAUDE.md. Blocks non-code files (MEMORY.md), files outside the project (~/.mcp.json), and even TRACES.md itself (the file the Stop hook demands you update).

**Root cause:** v1.0-beta already replaced the prompt-type hook with a command-type script (`check-sequential-files.sh`) that never blocks (always exit 0). But the script has a bug — it checks `agent_type` which doesn't exist in the hook input JSON schema.

**Fix:**
1. Remove the `agent_type` check from `check-sequential-files.sh`. The script should fire for ALL callers (main + subagent) — the warning is advisory anyway (never blocks).
2. Alternatively, check `session_id` or another available field to distinguish subagents if we want the "main session can edit anything" behavior. Need to verify what fields actually exist in hook input.

**Files to modify:** Skill file step 7a (`check-sequential-files.sh` template)

---

### Issue 2: `stop-check.sh` File Exclusion Pattern Bug [HIGH]

**Problem:** Line 412 uses `grep -vE '(\.md |\.txt )'` with a trailing space to exclude markdown/text files. But `git status --porcelain` puts filenames at end of line — no trailing space. The pattern never matches, so `.md` and `.txt` files are counted as "code changes," causing false Stop hook triggers.

**Fix:** Change pattern to use `$` anchor:
```bash
# Before (broken):
grep -vE '(\.md |\.txt )'
# After (fixed):
grep -vE '\.(md|txt)$'
```

**Files to modify:** Skill file step 7b (`stop-check.sh` template)

### Issue 3: Stop Hook Fires Too Often [HIGH]

**Problem:** The Stop hook fired 8 times in a 41-minute session. The v1.0-beta already converted it from prompt-type to command-type with `stop_hook_active` guard and exit 2 + stderr. However, the audit was done against the v0.9-alpha prompt-type hook. Need to verify the current command-type implementation actually solves the firing frequency issue.

**Analysis:** The v1.0-beta command-type `stop-check.sh` with `stop_hook_active` guard should already fix this — it exits 0 immediately on re-fire. The audit predates this fix. **No further changes needed** unless testing reveals the command-type hook still has frequency issues.

**Action:** Verify in testing. No skill change needed for this issue specifically.

### Issue 4: Build Roadmap Never Gets Updated [HIGH — NEW FEATURE]

**Problem:** TRACES.md works because it's local (readable, writable, enforceable by hooks). Build Roadmap lives only in Notion — behind an MCP call, invisible to hooks, easily forgotten. The enforcement gap analysis showed: Stop hook can mention Roadmap in stderr, but shell hooks can't verify Notion state.

**Solution:** Local `ROADMAP.md` with checkout/commit model.

**Architecture:**
- **Session start** → query Notion → generate `ROADMAP.md` (sync down)
- **During session** → Claude works from `ROADMAP.md` (local state)
- **State transitions** → update `ROADMAP.md` + call `notion-update-page` (sync up)
- **Notion offline** → local file still works, Claude notes "sync pending"
- **Conflict resolution** → session start always overwrites local from Notion (one-direction-at-a-time)

**Format:**
```markdown
# Build Roadmap

Last synced: 2026-03-10T14:30:00Z | Sprint: 3

## In Progress
- Implement OAuth2 login flow [P0, feat/auth-flow] <!-- notion:page-id-123 -->

## Up Next
- Add rate limiting to API [P1] <!-- notion:page-id-789 -->

## Verifying
- Fix mobile header overlap [Sprint 2] <!-- notion:page-id-345 -->

## Insights
- Database indexes needed for search queries <!-- notion:page-id-678 -->
```

**Changes required:**

#### 4a. New step in skill: Create ROADMAP.md template (after Step 5)
Add a step that creates `ROADMAP.md` in the project root with a template. If Notion is connected, populate it from the database. If not, create empty template.

#### 4b. Update CLAUDE.md Build Roadmap section
Replace the current "Reading the Roadmap" instructions (which tell Claude to call `notion-query-database-view`) with the checkout/commit model:
- Session start: rebuild `ROADMAP.md` from Notion (if connected)
- During work: read/update `ROADMAP.md` directly
- State transitions: update `ROADMAP.md` AND call Notion API in the same action
- If Notion unavailable: update `ROADMAP.md` with "sync pending" note

#### 4c. Update SessionStart hook (startup)
Add `ROADMAP.md` to the startup directive: "Read TRACES.md ... (4) Rebuild ROADMAP.md from Notion if connected, or read existing ROADMAP.md"

#### 4d. Update Stop hook (`stop-check.sh`)
Add `ROADMAP.md` modification time check alongside TRACES.md check. If code was modified but neither TRACES.md nor ROADMAP.md was updated, fire the reminder.

#### 4e. Add ROADMAP.md to .gitignore recommendation
`ROADMAP.md` is ephemeral working state with Notion page IDs — should not be committed. Add a note in the skill to recommend adding it to `.gitignore`.

### Issue 5: Hook Conflict Loop [MEDIUM]

**Problem:** Stop hook demands TRACES.md update → PreToolUse blocks TRACES.md edit → retry triggers Stop again.

**Analysis:** This was caused by the v0.9-alpha prompt-type PreToolUse hook. The v1.0-beta command-type `check-sequential-files.sh` never blocks (always exit 0), so the conflict loop should already be resolved. However, we should ensure TRACES.md and LEARNINGS.md are explicitly excluded from sequential file checks.

**Fix:** Add a comment in the `check-sequential-files.sh` template noting that TRACES.md and LEARNINGS.md should never be in `sequential-files.txt`. Add an early exit if the file being edited is TRACES.md or LEARNINGS.md (belt and suspenders).

**Files to modify:** Skill file step 7a

---

### Issue 6: Protocol Scope Ambiguity [LOW]

**Problem:** Branch lifecycle unclear for config-file-only projects. Infrastructure setup (TRACES.md, hooks) unclear re: needing a Roadmap item first.

**Fix:** Add explicit scope clarifications to the CLAUDE.md Branch Lifecycle section:
- Already partially addressed — the section says "Does NOT apply to: Build system setup, Documentation-only changes, Global config files"
- Tighten the language: "Build system infrastructure (creating/updating TRACES.md, LEARNINGS.md, ROADMAP.md, CLAUDE.md build system section, hooks, settings.json) is exempt from both branch lifecycle and Roadmap requirements."

**Files to modify:** Skill file step 6 (CLAUDE.md template)

---

### Issue 7: `check-sequential-files.sh` Field Validation [LOW]

**Problem:** Uses `agent_type` field which may not exist in hook input. The v1.0-beta skill comments say it uses `agent_type` (not `agent_name`), but the audit found this doesn't work either.

**Fix:** Research what fields are actually available in PreToolUse hook input JSON. The documented fields are: `tool_name`, `tool_input`, `cwd`, `session_id`. Remove the subagent-only check entirely — the warning is advisory (never blocks), so it's harmless to show it to the main session too. Simpler = fewer bugs.

**Files to modify:** Skill file step 7a
